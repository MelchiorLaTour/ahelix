# Assignment Methodology

Published opposition pairings and rotation rules. Transparency in how agents are
assigned prevents ad hoc bias and enables external review.

## Core Principle

Every Believer must face a structurally opposed Analyst. Every Worker must have
an Analyst reviewer. Pairings are intentional, not random.

## Default Opposition Pairings (Believer ↔ Analyst)

```
BELIEVERS (3):                    OPPOSED ANALYSTS (6):
  Empiricist-Believer        ↔    Systems Thinker-Analyst
  Institutionalist-Believer  ↔    Risk Manager-Analyst
  Pragmatist-Believer        ↔    Economist-Analyst

  Remaining Analysts (roaming critics):
    Contrarian-Analyst   — challenges any position
    Futurist-Analyst     — challenges temporal assumptions
    Moralist-Analyst     — challenges ethical blind spots
```

## Worker ↔ Analyst Review Pairings

Each Analyst is assigned 1-2 Workers to review for flip rationality.
This distributes review load and prevents audit bottlenecks.

### 18-Agent Mode (Full)

```
Systems Thinker-Analyst  → reviews Workers 10, 11
Risk Manager-Analyst     → reviews Workers 12, 13
Economist-Analyst        → reviews Workers 14, 15
Contrarian-Analyst       → reviews Workers 16
Futurist-Analyst         → reviews Workers 17
Moralist-Analyst         → reviews Workers 18
```

### 15-Agent Mode (Simple)

Workers reduced from 9 to 6. Each Analyst reviews 1 Worker.

```
Systems Thinker-Analyst  → reviews Worker 10
Risk Manager-Analyst     → reviews Worker 11
Economist-Analyst        → reviews Worker 12
Contrarian-Analyst       → reviews Worker 13
Futurist-Analyst         → reviews Worker 14
Moralist-Analyst         → reviews Worker 15
```

## Rotation Rules by Topic Type

Pairings are not fixed forever. They rotate based on topic category to prevent
stale relationships and ensure fresh challenge dynamics.

| Topic Category | Believer Lead | Primary Opposition |
|---|---|---|
| Technical / Engineering | Empiricist | Systems Thinker |
| Policy / Governance | Institutionalist | Moralist |
| Business / Strategy | Pragmatist | Economist |
| Ethical / Values | Moralist (promoted) | Contrarian |
| Speculative / Future | Futurist (promoted) | Risk Manager |

When an Analyst is promoted to Believer role for a specific topic, a Worker with
the strongest relevant archetype is promoted to fill the Analyst gap.

## Archetype Selection Guidelines

Not every debate uses the same 18 archetypes. Selection depends on topic.

**Always include:**
- At least 1 Empiricist (data anchor)
- At least 1 Contrarian (premise challenger)
- At least 1 Affected Party in Workers (lived experience)

**Topic-dependent:**
- Technical topics → Insider, Systems Thinker, Risk Manager
- Social topics → Humanist, Affected Party, Outsider
- Strategic topics → Economist, Futurist, Pragmatist
- Ethical topics → Moralist, Historian, Devil's Advocate

## Learned Assignment (Ring 3+)

After Rings 1-2, agent performance data is available:

- **Rational flip rate**: How often did this agent flip for good reasons?
- **Engagement score**: Did the agent substantively engage or phone it in?
- **Coverage contribution**: Did the agent surface unique perspectives?

In Ring 3+, prioritize agents with high scores. Deprioritize agents who added
noise without signal. This is optimization, not punishment — an agent that
underperformed on one topic may excel on another.
